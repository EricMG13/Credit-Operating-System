# Nielsen usability scorecard

Scale: 0–4 per heuristic; 40 maximum. The worst supported desktop/tablet instance determines each score.

| # | Heuristic | Score | Evidence |
|---|---|---:|---|
| 1 | Visibility of system status | 4/4 | Completion axes, observed/as-of labels, persistent Reference marker, Pipeline planned/executed counts, and upload outcome states expose current status. |
| 2 | Match between system and the real world | 4/4 | Primary copy uses portfolio, issuer, evidence, dependency map, stage lanes, run activity, and QA review; CP codes are secondary notation. |
| 3 | User control and freedom | 4/4 | Context-preserving navigation, collapsible supporting panes, reversible previews, focus restoration, and explicit persist/decision actions avoid accidental commitment. |
| 4 | Consistency and standards | 4/4 | Canonical route registry, shared EnterprisePage/Panel/DecisionHeader, typed PageAction, semantic tokens, and shared table/numeric rules. |
| 5 | Error prevention | 4/4 | Immutable gates, unavailable reasons, upload readiness prerequisites, navigation guards, typed action effects, and live/reference isolation prevent false commitments. |
| 6 | Recognition rather than recall | 4/4 | Labelled priority rail/drawer, current workflow, task-language tabs, visible units, grouped Deep-Dive layers, and ordered Pipeline peer table. |
| 7 | Flexibility and efficiency of use | 4/4 | Role-priority composition, keyboard roving, Alt+K, command palette, table presets, compact PM summaries, and QA-leading governance support expert workflows. |
| 8 | Aesthetic and minimalist design | 3/4 | Sparse frames and duplicate affordances were removed, but Model/Portfolio retain high initial structural/resource density as an explicit refinement watch. |
| 9 | Help users recognize, diagnose, and recover from errors | 4/4 | SurfaceState, ActionReason, outcome-specific upload states, offline/unavailable distinctions, validation copy, and retry/reopen paths state cause and next action. |
| 10 | Help and documentation | 4/4 | Advanced capability details, limitations disclosures, keyboard instructions, evidence summaries, formula/source explanations, and task-specific empty states are available in context. |
