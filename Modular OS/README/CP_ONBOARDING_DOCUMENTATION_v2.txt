# CP Agents — Credit Analysis Co-Pilot
# Onboarding Documentation
# Version: 2.0 | Date: 2026-06-08

---

## 1. Purpose and Scope

CP Agents is a modular, AI-driven credit analysis operating system designed to augment credit analysts in producing institutional-quality credit research. The system comprises **27 specialised agents** — 24 analytical modules organised into 8 layers, plus 3 infrastructure services — that work together to deliver deep, structured, evidence-based credit analysis.

### What the system does

- Ingests and validates source documents before analysis begins
- Routes execution intelligently based on task type and available inputs
- Produces structured issuer profiles, business risk assessments, and credit narratives
- Performs detailed financial synthesis, EBITDA quality review, and addback challenge
- Maps full capital structures, maturity profiles, and refinancing/LME pathways
- Analyses covenant packages and calculates compliance capacity
- Generates relative value assessments and market context
- Conducts adversarial bull/bear/IC chair credit debates
- Produces portfolio-level allocation challenges and sizing recommendations
- Produces sector-level credit reviews with comparative peer tables and early warning dashboards
- Monitors credit portfolios continuously via the CreditPulse alert system
- Integrates email intelligence (rating actions, sell-side research, trading desk color, internal analyst notes) as a formal source tier across all modules
- Renders final output in multiple formats (markdown, JSON, Word)

## 5. How to Get Started

1. Upload this onboarding document and the governance files to your Copilot session.
2. Review `MODULES_REFERENCE.md` to understand each module's purpose and inputs.
3. Review `MODULE_EXECUTION_ORDER.md` to understand execution sequencing.
4. Review `EXAMPLE_PATHWAYS.md` for common workflow patterns.
5. Start with CP-0 (Source Readiness) to validate your document pack.
6. Use CP-X to route to the appropriate analytical module.
7. Follow the suggested prompts in `SUGGESTED_PROMPTS.md` for each module.
8. Use CP-SR for sector-level credit reviews with email intelligence integration.
9. Configure CP-MON for continuous monitoring and alert generation across your coverage universe.
10. Upload `REF_CP-EMAIL_SourceRoutingMatrix.md` to enable standardised email integration across all modules.

## 8. Common Errors and Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| Module returns generic output | Missing REF files or Schema | Re-upload all module files |
| CP-X routes to wrong module | Ambiguous task description | Use explicit module ID in prompt |
| CP-0 flags source gaps | Incomplete document pack | Upload missing sources |
| Financial metrics disagree with source | Stale or conflicting data | Check data-as-of dates per source |
| Covenant capacity returns N/A | Missing credit agreement | Upload indenture or credit agreement |
| CP-6A debate is one-sided | Insufficient bear/bull evidence | Provide rating reports and sell-side research |
| CP-SR returns Low confidence | Insufficient Tier 1/2 sources for sector | Provide additional source documents or enable email intelligence intake |
| CP-MON not triggering alerts | Email classification not matching sector | Review REF_CP-SR_G email taxonomy and verify sector keywords in runtime config |

## 9. Email Intelligence Integration

### Overview

`REF_CP-EMAIL_SourceRoutingMatrix.md` is the cross-module governance file that standardises how email-derived intelligence is routed to all CP modules. It defines permissions, confidence treatment, staleness rules, and guardrails for each module.

### Use Types

| Use Type | Definition |
|---|---|
| Evidence | Email content directly supports a conclusion (subject to tier and confidence rules) |
| Context | Email enriches interpretation but does not independently prove the conclusion |
| Trigger | Email initiates a module run, refresh, or alert |
| Routing Signal | Email tells CP-X which module should run next |

### Email Categories

| Category | Default Tier |
|---|---|
| RATING_ACTION | 1.5 |
| INTERNAL_RESEARCH | 1.5 |
| SELL_SIDE_RESEARCH | 2.0 |
| SELL_SIDE_NEWS | 2.0 |
| TRADING_DESK | 2.5 |
| MARKET_DATA | 2.5 |
| EVENT_INVITE | 3.0 |
| INTERNAL_COMMS | 3.0 |

### Source Tier Hierarchy

| Tier | Reliability | Examples |
|---|---|---|
| 1 | High | Company filings, OMs, credit agreements |
| 1.5 | High | Internal analyst notes, forwarded rating actions |
| 2 | Medium-High | Agency reports, CreditSights, bank research |
| 2.5 | Medium | Trading desk color, pricing axes |
| 3 | Medium | Industry associations, media, market commentary |
| 4 | Low | Social media, unverified estimates |

### Rollout Priority

| Phase | Modules | Rationale |
|---|---|---|
| Phase 1 | CP-MON, CP-1, CP-3/3C/3D, CP-5/5B, CP-SR | Highest real-time credit intelligence benefit |
| Phase 2 | CP-6A, CP-6E, CP-X, CP-2/2E | Analytical enrichment with guardrails |
| Phase 3 | CP-4/4C, Legal/Structural, Reporting | Triggers, context, and evidence appendix only |
