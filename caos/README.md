# CAOS — Credit Agent OS

Credit analysis workspace: a six-concept analyst UI (Command Center,
Pipeline, Deep-Dive, Model Builder, Report Studio, Monitor) plus a global
**Ask (⌘K)** launcher, backed by a FastAPI service. Built to deploy as a single
**Databricks App** — one process serves the JSON API under `/api` and the
statically-exported Next.js frontend at `/`.

```
caos/
  server/      FastAPI app (this is the Databricks App source dir)
    app.yaml   Databricks Apps entrypoint
    static/    Built frontend (generated by scripts/build_frontend.sh)
  frontend/    Next.js 15 app, output: "export"
  scripts/     build_frontend.sh
  tests/       server API tests + Playwright e2e
  databricks.yml  Asset Bundle definition
```

## Architecture on Databricks

| Concern        | Implementation                                                       |
|----------------|----------------------------------------------------------------------|
| Auth           | Platform-managed (workspace OAuth at the edge). The app reads the forwarded identity headers (`X-Forwarded-User` / `-Email`); there is no in-app login. |
| Database       | SQLite by default (ephemeral); set `DATABASE_URL` to Lakebase (`postgresql+asyncpg://…`) for durability. Schema is managed by **Alembic** (migrations run on boot). |
| Document vault | Local dir by default; set `CAOS_STORAGE_DIR` to a Unity Catalog Volume (`/Volumes/...`) for durability. |
| LLM chat       | Anthropic Claude (`claude-opus-4-8`) via `ANTHROPIC_API_KEY` (Databricks secret). Without a key the chat endpoint returns canned demo replies. |
| Frontend       | Next.js static export served by FastAPI's StaticFiles — no Node process in deployment. |

## Deploy to Databricks

Prereqs: [Databricks CLI](https://docs.databricks.com/dev-tools/cli/) ≥ 0.230
authenticated to your workspace, Node 20+ locally to build the frontend.

```bash
cd caos

# 1. Build the UI into the server bundle
./scripts/build_frontend.sh

# 2. Deploy the bundle (creates/updates the app)
databricks bundle deploy -t dev

# 3. Start the app
databricks bundle run caos -t dev
```

Then open the app URL shown in the workspace (Compute → Apps → caos).

Optional durability/LLM wiring (edit `server/app.yaml` env entries):

- `DATABASE_URL` → a Lakebase database instance
- `CAOS_STORAGE_DIR` → a UC Volume path for uploaded documents
- `ANTHROPIC_API_KEY` → reference a Databricks secret to enable live chat

## Local development

Two options:

**Single process (what production runs):**

```bash
python3 -m venv .venv && .venv/bin/pip install -r server/requirements.txt
./scripts/build_frontend.sh
cd server && ../.venv/bin/python run.py     # http://localhost:8000
```

**Hot reload (UI dev):** run the API and `next dev` side by side — the dev
server proxies `/api` to :8000.

```bash
cd server && ../.venv/bin/python run.py &
cd frontend && npm install && npm run dev   # http://localhost:3000
```

## Tests

```bash
# Server API tests
.venv/bin/pip install pytest httpx
.venv/bin/python -m pytest tests/server -q

# Frontend unit tests
cd frontend && npm test

# E2E (needs the single-process server running on :8000)
cd frontend && npm run test:e2e
```
