## C3 — is_finite_number guard (P0). CHOSEN: `safe_div` in periods.py.

- Blast: periods is fan-in 20 = HIGH by definition, but this ADDS functions (no interface change to existing `is_finite_number`/`latest`/`sort_key`), so additive/low-risk. Goldens must stay green (identical results).
- P0 justification: it is THE guard law made structural. NOTE: verify (W2) whether any CP-1 divide is currently UNguarded — a live one = P0 bug, not just hardening. All divides I read (downside/metrics/adjusted/metricengine/peers/schemas) are guarded.
