# Path to Production — spec working notes

Working memory for authoring `PATH_TO_PRODUCTION_SPEC.md` (per brief §0.1). Records the
pattern chosen per layer, candidates rejected as redundant with existing posture, and
the fixed boundaries every item is checked against. Not a deliverable.

## Fixed boundaries (brief §0) — every item must survive all five
1. No added latency on interactive `/api` path (metrics in-memory + scrape-pull, correlation O(1), breaker = in-memory state read, tracing cheap/sampled).
2. No behavior change (analytical output, API contracts, mock↔engine seam byte-identical).
3. Single-process, no-paid-services (OSS/self-hosted only; no broker, no 2nd runtime; DB-lease executor is the concurrency substrate).
4. Preserve/extend fail-closed guards (`main.py` lifespan, `config.py` `is_deployed`); new config fails toward enforcement.
5. Spec, not code (real paths + named blocks, no line numbers).

## Do-NOT-re-spec (brief §3 strong posture)
Fail-closed boot guards; `:?`-required compose injection; structured `caos.access` + `caos.llm` streams; `sanitize_field`; `log_unhandled`; DB claim safety (`with_for_update(skip_locked=True)`, lease + `_reap_orphans`, boot sweep, advisory-lock-serialized migrations); `pool_pre_ping=True`; timeouts everywhere (`caos_llm_timeout_s`, `edgar_timeout_s`, `clamav_timeout_s`); graceful degrade (rerank/embeddings fail-open, ClamAV fail-closed, LLM fallback swap); health readiness probe; container hardening; no secret leakage.

## Per-layer pattern decisions (fill as authored)

### L0 — Config & Secrets
- **Chosen:** pydantic `SecretStr` for the 6 secrets; `*_FILE` file-based injection (Docker/K8s secret-mount convention) with env fallback, resolved in `config.py`; boot-time flag-prerequisite audit (warn-only, never block — degrade-to-safe already exists); dual-key `session_secret` (verify-list, sign-with-primary) so rotation ≠ mass logout; rotation runbook doc.
- **Rejected:** Vault/cloud secret manager (violates no-paid-services + single-node); flipping flag-audit to fail-closed (would break the existing degrade-to-safe contract → behavior change).

### L1 — Observability Substrate
- **Chosen:** stdlib `logging.Formatter` subclass emitting single-line JSON (NO new dep — a formatter subclass replaces `basicConfig`, `caos.access`/`caos.llm` keep emitting their JSON payloads which the formatter passes through); `contextvars` request-ID (honor inbound `X-Request-Id` from edge else generate), set in an ASGI middleware, injected as a log field via the formatter; JOIN (not replace) the existing `budget._run_id_var`.
- **Rejected:** structlog / loguru (new dep for what a Formatter subclass does); replacing `run_id` (brief: extend, not replace).

### L2 — Network & Edge
- **Chosen:** request-ID origination at Caddy (`header_up X-Request-Id {http.request.uuid}` or equivalent, forwarded through oauth2-proxy); compose healthchecks for oauth2-proxy + caddy ONLY (db/clamav already have them, app's is the Dockerfile HEALTHCHECK — no duplicate); liveness endpoint (no-DB, process-up) split from the existing DB-touching readiness probe.
- **Rejected:** adding an app compose healthcheck (redundant with Dockerfile HEALTHCHECK); replacing the readiness probe (keep it, add liveness beside it).

### L3 — Data & Persistence
- **Chosen:** explicit `pool_size`/`max_overflow` coupled to `caos_run_concurrency` + `synth_concurrency`; `pool_recycle`, `pool_timeout`, connect timeout; server-side `statement_timeout` via connect args / `SET` (Postgres only, guarded off SQLite); transient-disconnect classification policy beyond `pool_pre_ping`. New knobs in `config.py`.
- **Rejected:** external pooler (PgBouncer) — 2nd runtime, single-node overkill; touching the strong claim/lease/reap path.

### L4 — Egress Resilience (FINAL)
- **Chosen:** REUSE the existing `_call_with_retry` (base1/max8/±10% jitter/max_retries=3, gated on `is_overloaded`) — generalize it to wrap the PRIMARY attempt + the openrouter/gemini provider branches (currently wraps only the fallback). Hand-rolled per-provider circuit breaker (~40 lines, in-memory closed/open/half-open, dict read = latency-trivial) at `create()`; ship a state-machine unit test. EDGAR: hand-rolled bounded retry honoring 429/503 `Retry-After` (custom wait — tenacity's wait_exponential can't express Retry-After) at `_http_get`; token bucket cross-process-safe via `engine/locks.py` advisory_lock + `key_from_str("edgar-fetch")`, `_MIN_INTERVAL_S` kept as single-process fallback. Cover the 3 bypass lanes (deepresearch `_final_message`, research_report `synthesize_research_report`, synth `_call`) via the shared retry/breaker helper. Match degrade per call site (embeddings/rerank fail-open, ClamAV fail-closed).
- **NO new dependency for L4.** tenacity NOT promoted (reused `_call_with_retry` instead — extend-before-add); pybreaker NOT added (hand-rolled breaker). Both named as available alternatives in-spec with rationale.
- **DRIFT corrected vs brief §3B:** Anthropic fallback = `fallback_model or s.synth_executor_model` (NOT `model_tier_cheap`; that's the openrouter/gemini target). Synth semaphore lives in `engine/runner.py` (`_attempt_synth`), NOT `synth.py`.
- **Rejected:** flipping any fail-open/closed choice; rerouting the 3 bypass lanes through `create()` (would change their streaming/tool semantics — share the helper instead).

### L5 — App Logic & Async Execution
- **Chosen:** trace span across request→executor→LLM arc keyed on the existing `run_id` (+ request-id from L1); job claim/start/heartbeat structured log events (hung-vs-crashed detection) on the 4 executor loops; per-run correlation binding at `execute_run`.
- **Open (decide in-execution):** tracing backend — OTel SDK + local collector (opt-in compose profile) vs span-shaped JSON log events on the L1 substrate. Present both; lean OTel-optional so zero cost when the collector is absent.
- **Rejected:** a message broker / distributed tracer requiring a 2nd runtime (boundary 3).

### L6 — Telemetry & Metrics
- **Chosen:** `prometheus-client` `/metrics` pull-scrape (in-memory counters/histograms, zero synchronous push per request); RED (rate/errors/duration) sourced from the access-log middleware seam, USE (queue depth/in-flight/failure) from the executor loops, LLM cost/fallback/latency derived from `budget.trace_llm` + `LLMCallRecord`; optional local Prometheus+Grafana compose profile (opt-in, not default).
- **Rejected:** OTel metrics SDK (heavier; prometheus-client is the lighter pull-only fit for single-node); any paid APM (boundary 3); synchronous metric push on the request path (boundary 1).

## Prometheus/OTel dep decisions (FINAL)
- `prometheus-client` = the ONE new dependency (L6, pull-scrape /metrics, in-memory, no collector required to emit). Justified vs no-paid-services.
- OpenTelemetry = OPT-IN only (L5 tracing upgrade, flag `caos_tracing_enabled=False`, SDK not imported when off). Default tracing = span-shaped JSON log events on the L1 substrate (zero dep).
