# PDF → OKF ingestion — architect working notes

Decision log for `PDF_INGESTION_OKF_BLUEPRINT.md`. Keeps the six stages internally
consistent and gives the §7 checkpoints something to audit against. Every decision
is grounded in a read/grep of the real code (paths inline). Not a deliverable —
scratch memory.

## The one-paragraph shape

Inbound external PDF credit reports become (a) `document_chunks` rows on the SAME
lane the existing uploader uses — so `retrieval.retrieve*` sees them with zero
forks — and (b) one Obsidian **Source note** per document under `Sources/`, a new
4th note family that projects structure + provenance + `[[wikilink]]` graph edges.
The knowledge graph is a derived projection keyed to `document_id`; canonical
facts stay in the engine DB. Five new flat modules + one route + one additive
table; no legacy schema alter, no UI edit, no paid vault embedder, no two-way sync.

## Grounded baseline (verified, not assumed)

- `ingest.chunk_text` is **token-based** (tiktoken `cl100k_base`, `max_tokens=512`,
  `overlap_tokens=64`, paragraph-aware). `CHUNK_CHARS=2400/CHUNK_OVERLAP=240` exist
  in `ingest.py` but are **unused by `chunk_text`** (brief §3 is wrong on this —
  blueprint states actual behavior). Reuse `chunk_text` as-is.
- Upload lane `routes/ingestion.py::_vault_document`: `Document` row → bulk
  `insert(DocumentChunk)` with explicit `chunk_hash=sha256(chunk)` + one
  `LineageEdge` per chunk → **background** `embed_chunks_for_document`. Zero chunks
  → Document persists `chunk_count=0`, `warning=ingest.NO_CHUNKS_WARNING`, HTTP 200.
- Retrieval (`retrieval.py`): BM25 (PG `tsv` generated col + GIN / SQLite Python
  Okapi ≤5000 rows) + vector, fused `rrf_fusion(k_rrf=60)`. **Every vector query
  hard-filters `DocumentChunkEmbedding.model == settings.embedding_model`
  (`"text-embedding-004"`, `SafeVector(768)`), INNER-joined on `chunk_hash`.**
  Foreign-model or non-768 vectors are vector-invisible; a chunk with no embedding
  row is BM25-retrievable, vector-invisible.
- Issuer scoping is a `Document` FK join; `retrieval.build_issuer_index` excludes
  `Document.doc_type != "analyst-memo"` — so `doc_type` is already a prefilter and
  the 5 new source types are (correctly) engine-visible.
- `Document.fiscal_period String(64)` already exists nullable — period prefilter
  without a schema alter. Nothing writes it today.
- Memo lane (`engine/memochunks.py`) is the inbound precedent: one
  `Document(doc_type="analyst-memo", storage_key=f"memo:{title}")` per issuer,
  reuses `ingest.chunk_text`, supersede via `_delete_prior_memo_docs`.
- `vault_export._yaml_block` emits `{k}: {json.dumps(v)}`, **drops None/""**, flat
  scalars only. Three existing note families: `credit-run` (spoke), `issuer` (hub),
  `analyst-memo` (memo). `_title` sanitizer is traversal-safe. `autolink_issuers`
  wraps first mention. `sync_analyst_memos` scans the whole vault **except
  `Runs/`+`Issuers/`** and rewrites `AnalystLink` — a `Sources/` tree must be added
  to that exclusion or OKF notes get ingested as analyst memos.
- No OKF, no structured-PDF extractor, no Ollama client exists today.

## Decisions

**D1 — Embedding backend.** Reuse `engine.embeddings.embed_chunks_for_document` on
the existing engine lane (`model=settings.embedding_model`, 768-dim). Only zero-fork
option (model hard-filter). Compliance: inbound chunks ride the same lane every
uploaded PDF already rides (Gemini w/ key; deterministic local `get_mock_embedding`
w/o). The **vault-file lane ships no CAOS embedder** — vault-side semantic stays
analyst-side local (Smart Connections/Ollama), per OBSIDIAN_DATABANK.md. No paid
service on the vault lane by construction. *(Model-flip is rejected not because it
orphans vectors permanently — `warmup_embeddings_task` re-embeds the corpus — but
because it forces a full re-embed spike; the reuse conclusion stands.)*

**D2 — Two write targets, one source of truth.** Machine-RAG corpus =
`document_chunks` via the existing write pattern (BM25 on commit; vectors via
background embed + startup warmup). Vault = knowledge-graph projection
(`Sources/` notes), regenerable from the stored blob, keyed to `document_id`.
Canonical facts remain in the DB; the note never feeds back.

**D3 — Prefilter metadata.** `Document.issuer_id` + `Document.doc_type` (existing,
existing filter-then-scan house pattern) + `Document.fiscal_period`. Richer OKF
fields (agency, ratings, report_date) live in frontmatter + the additive registry.

**D4 — Chunking: section-aware wrapper over `ingest.chunk_text`.** Note body = full
structured source text under `##` sections (page anchors in the heading). Per
section: prepend a **numeral-lean** breadcrumb (`{Issuer} — {doc_type} — {section}`,
no bare page integers → doesn't widen `grounding.all_grounded`), then run
`ingest.chunk_text` within the section. Chunk only the rendered BODY; skip empty
sections (zero-text PDF → zero chunks, not boilerplate). Tables stay contiguous in
their section (no table-parser dependency). One chunk ↔ one section ↔ one page
range → one coherent-idea embedding + a clean citation lineage line.

**D5 — Redaction posture (inbound family).** Source notes carry extracted source
text BY DESIGN; frontmatter `contains_source_text: true` flags them for
sync-exclusion tooling. `_redact` stays untouched for the outbound engine-output
families. Posture stated in the OKF spec + OBSIDIAN_DATABANK.md update (D11d).

**D6 — Structuring: deterministic core.** Doc-type classifier (first-pages markers),
section segmentation (markitdown headings when on; regex heuristics otherwise),
typed `KeyFact` extractors (ratings, tranche amounts, maturities, leverage). LLM
enrichment deferred as a named future lane (house fault-isolation: Blocked-gate /
return_exceptions / deterministic fallback = section-only). `issuer_id` required at
intake (matches the existing uploader).

**D7 — Degradation, never silent.** Zero-chunk PDF → note still written,
`extraction_status: empty`, registry flagged, `NO_CHUNKS_WARNING` surfaced. Partial
extraction → well-formed note + `extraction_status: partial` + warnings. Extraction
runs fully (via `to_thread`) BEFORE the write transaction opens — the typed payload
is the only thing crossing the seam.

**D8 — Doc-type vocabulary.** `rating-report | offering-memo | sponsor-deck |
lender-update | source-document` (fallback), stored verbatim in both
`Document.doc_type` and frontmatter `doc_type`. No collision with existing values.
`engine.readiness._categorize` substrings extended additively (D11c).

**D9 — Vault placement.** `{vault_export_dir}/Sources/{_title(f"{issuer} - {doc_type}
- {source?} - {report_date}")}.md`, family `type: "source-document"`, overwrite in
place (filename = supersede identity → stable across re-ingest). `document_id` in
frontmatter, never the filename. Registry `note_path` UNIQUE; supersede unlinks the
prior file on slug drift; residual collision → deterministic ` - 2`. Inbound NEVER
touches `Runs/`/`Issuers/`/`Analyst-Memos/`; `[[Issuer]]` backlink gives the graph
edge even before a hub note exists (memo precedent). `sync_analyst_memos` exclusion
gains `Sources/`. Lane gated on `vault_export_dir` (503 otherwise — memo precedent).

**D10 — Idempotency / supersede (the 3-HIGH fix).**
- Identity key `(issuer_id, doc_type, source?, report_date-or-fiscal_period)`.
  Per-issuer `content_sha256` match = true-duplicate no-op → return existing
  `document_id`.
- Evidence-aware: replace only if the prior doc's chunks have NO
  `EvidenceItem.document_chunk_id` reference; else keep old Document+chunks as a
  shadow and repoint the registry (preserves citations). Shadow chunks stay
  retrievable until the named upgrade path — rare (same-dated corrected re-upload
  after a citing run); stated as a known limitation.
- No legacy alter (no `documents.superseded_at`); supersede state lives in
  `okf_notes` only.
- Txn order: resolve identity → sha no-op check → insert new Document+chunks+
  LineageEdge → UPDATE okf_notes row in place → evidence-aware cleanup of prior doc
  → commit → background note write → background embed. Repoint-before-delete = no FK
  violation. Blueprint states the PG-enforces / SQLite-ignores-`PRAGMA foreign_keys`
  asymmetry and mandates the supersede test assert PG semantics.
- Storage blobs retained on supersede (audit + regeneration).

**D11 — New surface (flat modules, house style).**
- `okf_schema.py` — pydantic: `ExtractedDocument`, `PageText`, `DocSection`,
  `KeyFact`, `StructuredReport`, `OKFNote`, `OKFChunk`; doc-type enum; `OKF_VERSION`.
- `okf_structure.py` — classifier + segmentation + typed extractors (deterministic).
- `okf_notes.py` — `render_okf_note` (imports `_yaml_block`, `autolink_issuers`,
  `_title` from `vault_export`), `okf_note_title`, `write_okf_note` (Sources/).
- `okf_ingest.py` — orchestrator (extraction wrapper → chunker → write stage).
- `routes/okf.py` — `POST /api/okf/ingest` (multipart), `GET /api/okf/documents`.
- migration `00XX_okf_notes` — additive table.
- (a) route copies the `upload_document` security ladder verbatim
  (`Depends(get_identity)` + `_upload_rate_guard` + `read_capped`→`sniff_pdf`→
  `avscan.scan`-before-parse + `uploaded_by=caller.email` + `main.py`
  `include_router`). (b) note write in a `BackgroundTasks` task w/ own
  `AsyncSessionLocal`, status `pending_note→written|note_failed`. (c)
  `readiness._categorize` substrings extended. (d) OBSIDIAN_DATABANK.md updated.
  (e) `Document.fiscal_period` canonical, registry copy = audit. (f) model-flip
  rationale corrected (re-embed spike, not orphaning).
- Zero new settings (reuse `vault_export_dir`, `caos_storage_dir`, embed/ocr/av).
- Tests `tests/server/test_okf_*.py`: fixture PDF → OKFNote shape → chunk
  BM25-retrievable → supersede idempotency incl. PG-FK semantics → Sources/ scan
  exclusion.

**D12 — Page map.** Thin `extract_pdf_pages(content)` (pypdf per-page, additive,
same lib already imported) for page anchors. markitdown/OCR paths lack a page map →
anchors omitted, `extraction_status: partial`; pages are never fabricated.
